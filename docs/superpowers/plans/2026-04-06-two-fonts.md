# Two-Font Typography Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove pixel/retro fonts entirely, leaving only Archivo Black, Space Grotesk, and Space Mono across tokens, globals, and every feature.

**Architecture:** Delete the unused font tokens/utilities, sweep the repo for `font-family` overrides, and enforce the policy via docs + stylelint. Use existing `.retro-*` helpers so components inherit typography instead of inlining `font-family`.

**Tech Stack:** Vite + React, CSS Modules, clsx, Stylelint, pnpm.

---

### Task 1: Token & Global Utility Cleanup

**Files:**
- Modify: `src/styles/tokens.css`
- Modify: `src/styles/globals.css`

- [ ] **Step 1: Remove retired tokens**
  - Delete lines defining `--font-pixel`, `--font-retro`, and any aliases (`--font-retro`, `--font-pixel` references in theme blocks).
  - Ensure only:
    ```css
    --font-display: 'Archivo Black', 'Space Grotesk', system-ui, sans-serif;
    --font-body: 'Space Grotesk', system-ui, -apple-system, sans-serif;
    --font-mono: 'Space Mono', monospace;
    ```
    remain under the Fonts section.

- [ ] **Step 2: Drop pixel/retro utilities**
  - Remove `.pixel-text` and `.retro-text` definitions, plus any helper that explicitly mentions `var(--font-pixel)` or `var(--font-retro)`.
  - Ensure remaining helpers reference the correct tokens, e.g.:
    ```css
    .retro-label {
      font-family: var(--font-mono);
      /* rest unchanged */
    }
    ```

- [ ] **Step 3: Verify globals**
  - Search `globals.css` for `font-pixel` / `font-retro` to confirm zero hits.
  - Save file.

### Task 2: Docs Update

**Files:**
- Modify: `docs/design-system.md`

- [ ] **Step 1: Update Typography section**
  - Replace the bullet list with explicit callouts:
    ```markdown
    - **Archivo Black (`--font-display`)** – hero headings, banners.
    - **Space Grotesk (`--font-body`)** – all body copy, buttons, links.
    - **Space Mono (`--font-mono`)** – labels, stats, tables.
    ```
  - Add a sentence stating pixel/retro fonts were removed and should not be reintroduced.

### Task 3: Feature & Shared Component Sweep

**Files:** All `src/**/*.module.css` and TSX files that set `font-family`.

- [ ] **Step 1: Build hit list**
  - Run `rg -l "font-family" src | sort > /tmp/font-files.txt`.
  - Review the list for any files that truly need overrides (likely none after helpers exist).

- [ ] **Step 2: Remove overrides referencing retired fonts**
  - For each file referencing `var(--font-pixel)` or `var(--font-retro)`:
    - Delete the `font-family` line.
    - If the selector needs special emphasis, compose or apply a helper class. Example CSS removal:
      ```diff
      - .rank { font-family: var(--font-pixel); }
      + .rank { /* inherits */ }
      ```
    - Update the corresponding TSX to add helper classes: `className={clsx(styles.rank, 'retro-label')}`.

- [ ] **Step 3: Normalize remaining overrides**
  - If a component hardcodes `font-family: var(--font-display)` or `var(--font-body)` unnecessarily, remove it and rely on helpers/parent inheritance unless the component absolutely requires it (rare).

- [ ] **Step 4: Shared components**
  - Ensure `src/shared/components/` (PixelBadge, PixelCard, Ticker, ErrorState, StatBar, etc.) use helpers or inherit, not raw `font-family` lines.
  - Update tests if they snapshot the presence of specific helper classes.

### Task 4: Stylelint Guardrail

**Files:**
- Modify: `stylelint.config.cjs` (or create if missing)
- Modify: `package.json` (lint script)

- [ ] **Step 1: Add disallowed rule**
  - In `stylelint.config.cjs`, add:
    ```js
    module.exports = {
      /* existing config */
      rules: {
        ...(module.exports.rules || {}),
        'declaration-property-value-disallowed-list': {
          'font-family': [
            /var\(--font-pixel\)/,
            /var\(--font-retro\)/,
          ],
        },
      },
    };
    ```

- [ ] **Step 2: Ensure lint script runs stylelint**
  - If not already, add a script: `"lint:css": "stylelint \"src/**/*.module.css\""`.
  - Update `pnpm lint` to run `pnpm lint:css && eslint src --max-warnings 0` (or integrate via package.json).

- [ ] **Step 3: Test guardrail**
  - Temporarily add `font-family: var(--font-pixel);` to a dummy file, run `pnpm lint:css`, and confirm it fails. Remove the dummy change.
